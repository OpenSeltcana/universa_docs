    universa v0.1.0 Universa.Database

    Allows for interaction with the database, both reading and writing.

    Creating database tables

    See Universa.Database.Table

    After which you can start making changes using this modue, for example:

    iex> alias Universa.Database
Universa.Database

iex> Database.run(fn -> 
  Database.write(%Database.NewObject{id: 20})
  Database.write(%Database.NewObject{id: 20, count: 2, test: :ok})
end)
:ok

    read(atom(), list()) :: map() | nil

    Retrieves the full version of object with matching primarykey (other properties are ignored) out of the database

    If object is found in database, will return its entirety, else will return empty.

    Examples 

    iex> alias Universa.Database
Universa.Database
iex> Database.run(fn -> Database.read(%Database.Entity{uuid: "test"}) end)
[{Database.Entity, "test", []}]
iex> Database.run(fn -> Database.read(%Database.Entity{uuid: "not_test"}) end)
[]
    Link to this function run(func) 
    run((() -> any())) :: any()

    Runs the func inside a :mnesia.transaction/2.

    Runs the entire code in one go, it ensures either all changes are made or none of them are made. Also allows multiple processes to change the same record without interfereing with another and prevents deadlocks by retrying only three times.

    Examples 

    iex> Database.run(fn ->
  Enum.each(Database.find(Database.Entity, []), fn(ent) ->
      Database.delete(ent)
  end)
end)
:ok
    Link to this function save() 
    save() :: :ok

    Stores all active tables on the disk, so if the server is restarted will use those values instead of empty.

    WARNING: also saves databases not part of Universa.

    Link to this function write(object) 
    write(map()) :: :ok

    Adds object to the database

    Will always return :ok regardless of wether an object existed with the same primary key or not. If another object has the same primary key, it will get overwritten.

Universa.Database
